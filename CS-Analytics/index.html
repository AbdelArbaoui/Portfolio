🤝 Fiches clients (CRM) partagé · équipe

Contacts, licences, suivi — partagé et **fusionné** entre collègues (la version la plus récente l'emporte).

Pourquoi ce découplage ?

À l'origine, un seul fichier gérait ces deux aspects → les collègues écrasaient mutuellement leurs favoris. Le découplage en **favoris (local)** et **fiches CRM (partagé)** a éliminé les conflits : chacun garde son portefeuille privé tout en partageant une base commune de fiches synchronisées par fusion de timestamps. Entièrement couvert par tests de non-régression.

Choix de Sécurité & mTLS

  • ✓Calculs locaux (On-Premise) — Tout le traitement de données, le parsing de logs et le rendu PPTX sont effectués localement. Aucune donnée n'est jamais envoyée vers le cloud, garantissant la conformité stricte avec les directives de cybersécurité.
  • ✓Secrets éphémères — Les mots de passe .p12 et tokens API ne sont jamais stockés ni loggués.
  • ✓mTLS sécurisé — Fichiers PEM temporaires extraits uniquement en mémoire puis supprimés dans un bloc finally strict.
  • ✓Sous-processus sécurisés — shell=False, arguments validés (noms de clients, dates) avant exécution.
  • ✓Calculs de KPI encadrés — Formules basées sur une liste blanche stricte de jetons, sans accès aux fonctions système.
  • ✓HTTPS obligatoire sur tous les appels d'infrastructure.

Fiabilité des Données

  • ✓Sauvegarde miroir — Copie de sauvegarde automatique de chaque configuration dans le profil utilisateur (anti-perte).
  • ✓Restauration intelligente — Un fichier vide ne peut jamais écraser un fichier contenant des données.
  • ✓Concurrent Merge — Fusion des fiches partagées par timestamp (la modification la plus récente l'emporte).
  • ✓Découplage local/partagé — Le portefeuille privé n'est jamais propagé sur le réseau.
  • ✓Résilience PPTX — Interception des fichiers verrouillés par PowerPoint avec fallbacks automatiques de sauvegarde.
